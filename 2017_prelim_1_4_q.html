Road Runner 1 Security Robot AI Development

Road Runner 1 is a remake Nexon did from an old game, Road Runner. The player wants to control the runner so that the runner doesn't get caught by the security robots and collect all the gold in the map.

Let us create a security robot AI so that it dcan get to the runner as quickly as possible. The AI needs to decide the order of actions it should take in order to get to the runner as quickly as possible given the current map status, the runner's location and the security robot's lovation. We will assume initially that the given security robot AI will always be standing on a block, and the runner is not moving.

The robot's movement rules are as follows:
          - The game map is designed with empty space, ladder, and blocks.
          - You cannot walk into a block.
          - If you're standing on a block, you can move either left or right.
          - On a ladder, you can also move up and down.
          - You can step on a ladder, and in such case you can go down the ladder.
          - If you're currently hanging on a ladder and the bottom is empty spacfe, you will fall below.
          - To move 1 step in either (left/right) direction, it takes H amount of time. (On a ladder, you can still move left/right and this will also take H time.)
          - To move 1 step up/down on a ladder takes V time.
          - To fall down one step takes time F.

Input

First line of input has the size of the map, N x M where N, M are integers between 1 and 3000 inclusive.
Second line has H, V, F with spaces in between respectively. H, V, F can be greater than or equal to 1 and less than equal to 10 9.
In the next M lines, we get the map information from top to bottom.
Empty sapce on a map is given as '.'
A block is given as 'B'.
A ladder is given as 'H'.
The location of the runner as 'R', and the robot as 'G'.

Output

On the first line, output the direction the robot should move in L, U, R, D without any space. You also need to output the fall. For instance, the above should output R, D. From the ladder, sliding to the right is R and the fall is D. If you fall multiple steps, you need to ouput D the number of steps you fall.
If there are multiple ways to get to the runner with the same minimum amount of time, you only need to output one of the possible answers. If it is impossible to get to the runner, output X.

Input Example 1

        5 5
        1 1 1 
        BBBBB
        B.R.B
        B.BHB
        BG.HB
        BBBBB

Output Example 1

        RRUUL

Warning

On the actual game, there are many extra factors such as there are digging blocks, holding on, etc. but we do not worry about this in this particular problem.

Marks Guideline

The input cases have 3 types: To get points, you need to have correct outputs for all test cases for one type of input test cases.
          - Type 1 (90 marks): In the given map, we do not get a ladder.
          - Type 2 (60 marks): H = V = F = 1
          - Type 3 (150 marks): No other restrictions other than the ones listed in the problem.
