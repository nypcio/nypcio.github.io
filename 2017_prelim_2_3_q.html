Break the Bricks

"Break the Bricks", a new game Nexon is developing has the following rules:
      - Rectangular bricks are placed across N rows. Each row has a minimum of 0 bricks and a max of million bricks.
      - Number of bricks on each row and a sequence of numbers from 1 to N (no repetitive numbers) are given in the input.
      - It's your job to break the bricks if the number of bricks on a row matches the number of that specific element in the sequence. In other words, if we labelled the first element of the sequence as P(1), second elemnt as P(2),..., ith element as P(i), there should be exactly P(i) - 1 rows with bricks less than the number of bricks on the ith row.
To explain more in deatil, if we visit the row according to the sequence and count the number of bricks on these rows, the number of bricks should be increasing.
     - In the input, the number of bricks on each row could be the asme, but in the end, they should be all different.
     - You will get the highest mark if you can achieve this by breaking the smallest number of bricks.

However, with the same number of bricks and but a different brick placement sequence (1, 2, 4, 5, 3), there isn't a solution because by only breaking bricks, column 3 cannot have the second most bricks.

Given the number of column of bricks, number of bricks per column, and the brick sequence, output the minimum number of bricks that need to be broken if the number of bricks per column can match the given sequence.

Input

The first line of input has the number of columns of bricks, N, where N is greater than or equal to 1 and less than or equal to 200,000. The next line has N numbers, and reading from left to right, it indicates the number of bricks on each column. The max number of bricks can be from 0 to 1 billion. The following line has the sequence of numbers 1 to N with no repetition.

Output

Output the minimum number of bricks that need to be broken in order to achieve the desired status. If this is not possible, output -1. If no bricks need be broken, output 0.

Input Example 1

        5
        3 5 2 6 4
        1 3 2 5 4

Output Example 1

        4

Input Example 2

   5
   3 5 2 6 4
   1 2 4 5 3

Output Example 2

     -1

Marks Guideline

The input cases have 2 types: To get points, you need to have correct outputs for all test cases for one type of input test cases.
          - Type 1 (60 marks): N is less than or equal to 1000, and the maximum number of bricks that can be placed per column is 10,000.
          - Type 2 (140 marks): No other restrictions other than the ones listed in the problem.
