Spin And Slide: Change Made

Let us write code that will will be implemented in a virtual puzzle. The size of the puzzle board is N x N, and each tile is either empty or filled in with a 1 x 1 square with a specific colour.

= Original Board

This is the original board. If we rotate this by 90 degrees clockwise, the puzzle board will look like this:

If we allow the blocks to slide down, the puzzle board will look like this:

= SpinAndSlide(Original Board, 1)

Let us define SpinAndSlide as rotating the puzzle board by 90 degrees clockwise and sliding all the coloured tiles to the bottom of the board.

Our goal is to output the puzzle board once SpinAndSlide is applied multiple times onto a given board.

For instance, if we apply SpinAndSlide one more time onto the previous example board, the outcome will look like this:

= SpinAndSlide(Original Board, 2)

Input

First line of input is the size N of the puzzle board. N must be an integer between 1 and 100 inclusive.

Second line of input is M, the number of times you wish to call SpinAndSlide. M must be an integer between 0 and 100 inclusive.

Starting from the third line of input, the original board will be given in N lines.

Each line is made up of N characters and each character that represents a coloured tile is written in capitalised alphabet and if it’s empty, it is represented by ‘.’.

The original set up is always the case in which the blocks are slid to the bottom of the board.

Output

Output the status of the board after SpinAndSlide is executed M number of times across N lines.

Each line needs to be represented by N characters.

If the tile is coloured, the correct capitalised alphabet needs to be in place of the tile, otherwise if empty, ‘.’.

Input Example

 
       7
       2
       .......
       .......
       ...A...
       ...B...
       ..ACA..
       ..BBB..
       .AAAAA.

Output Example

       .......
       .......
       A......
       B......
       AAA....
       BBB....
       ACAAA..
