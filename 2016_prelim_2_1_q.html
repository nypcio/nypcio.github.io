How much did you walk?

To match the current trend of games, Joe wanted to develop a game that one could play while actually walking around.
In order to prevent users from driving in a car to increase their moving distance, Joe needs to figure out how to actually calculate the amount the user walked.
Given the time a certain movement was recorded and the distance in one day, develop a program that determines the distance that the user actually walked.
Here, we assume that the walking speed is between 3km and 5km inclusive. For speeds slower or faster than that range is not considered in the calculation.

Input

First line of input has the number of input data, N. The next N lines have the input data. Each line following the first line has 3 information: the time the movement started, finished time, and the distance travelled.
The times are given in a 24-hour system in hour:minute, and the distance is in meters. The distance is always a positive integer, and the speed never exceeds 1200km. (We exclude the situation when Joe takes a rocket or a plane.)
Arriving time and starting time can never be the same and there can never be overlap when one detects a movement. However, the end time of a movement and the start time of the next movement may be the same. The input will never have a situation where it goes over midnight so that it becomes a different day.

Output

Output the total distance that Joe travelled in the average speed in between 3km and 5km.

Input Example 1

          3
          09:35 10:01 401
          10:11 10:30 1361
          10:30 10:34 297

Output Example 1

        1658

The second and third movement was summed as the average speed was in between 3km and 5km. However, the first movement is 0.92km so it's not included.
