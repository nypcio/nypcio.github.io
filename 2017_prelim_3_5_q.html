Find the way all at once

We can calculate the shortest distance of an object moving to a specific destination on a 2-D coordinate plane using Dijkstra or A* algorithm. However, for a strategy simulation game, in the case multiple units are moving together and if it's not possible for two or more units to be in the same coordinate, these units become obstacles, making it harder for us to directly apply these algorithms.
Let us great a program that allows all units to move to its specific destinations as quick as possible under these conditions:

- In each unit, we are given a color. There can be multiple units with the same color, and for each color, the destinations are selected based on the units with the same color.
- In a unit, there can be a max of 2 colors.
- For every turn in the simulation, each unit can either move up down left right by one step or not move at all.
- For every turn in the simulation, a unit can only move to an empty location at its current time. Meaning, if a unit next to the unit you are trying to move moved on step does not mean you can move to that location in that turn.
- At each turn, preplanned moves are made all at once and that turn finishes.
- Two or more units cannot stay in one location together. Therefore, the starting points of the units are always in a different location, and two or more units cannot move to the same place.
- Everywhere on the map is connected other than some obstacle walls placed, and units cannot move out of the map.
- After all turns, if the unit places its color to one of its given destination points, then we say that unit has arrived to its destination.

Input

The first line has height H and width W. (1 ≤ H ≤ 100, 1 ≤ W ≤ 100). Across the next H lines, for each line we are given the map with W letters. The map letters are: #, ., A, B, a, and b.
These the rules on how to read the map:
    - # means an obstacle wall. The unit cannot move pass this.
    - . means an empty space. The unit can move pass this.
    - Captial letters such as A means that that color's unit is placed there. The total number of units will always be less than or equal to 100.
    - Lowercase letters such as a means that that color's unit's destination is there. The unit can get to this destination by moving through empty spaces.

For every test case, to show you how the input data generally looks like, we've provided you a file. The input file may not be in the exact format as shown in the files. Image files are as follows:
      - The unit's current location is either colored in blue/orange shape.
      - The unit's destination is either not filled in blue/orange shape.
      - The walls or obstacles are colored black.

      - On the first line, output the number of turns you wish to take (T).
      - For each turn, output N, the number of units you wish to move.
      - Across the next N lines, output .
      - For , the top most is 1 and bottom most is H.
      - For , the left most is 1 and right most is W.
      - can be U, D, L, R where U is up, D is down, L is left, and R is right.
      - The total number of movements cannot exceed 120,000.

Marks Guideline

For each test case, this is how you receive marks.
          - If you output something wrong, or if more than half of the unit did not arrive to the destination, then you do not receive marks.
          - We mark the same test input multiple times. In the case that your answer changes throughout these multiple tests, you do not receive any marks.
          - If you do not apply to the above situation, you receive mark penalties under the following rules:
                - Penalty = (number_units_that_didn't_arrive_to_dest) * 50 + (number_of_turns)
          - For each test case you receive: (full marks of the test case) * (B / T) where:
                - T = Penalty of that test case you received
                - B = the minimum penalty mark of all the contestants who solved this problem
Your total marks are your points added up from all test cases. By the way this question is marked, your score may vary continuously.
