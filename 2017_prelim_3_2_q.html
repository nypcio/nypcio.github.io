Match the Tiles

The world famous tile company, AwesomeTile, decided to produce interesting tiles with Korean cahracters as it is their 100th anniversary. Square tiles are in the form of a 3 x 3 grid, and as shown below, each grid has one of the four korean words: [풀], [물], [길], [흙].
We wish to design a map by adding several tiles. The example below shoes the map of 4 connected tiles. we say the file/grid are "adjacent" at the connection points of the tiles and individual grids.
Also, maps produced with the following rules are considered "Good maps" while if not, those maps are called "Bad maps".

The rules are as follows:
      1) [길] grids should always be adjacent within 1 tile.
      2) In a tile with at least one [길] grid, one of these [길] grids should be connected to at least one different tile with a [길] grid.
      3) [길] grid should alawys be in some form of a line. (2 x 2 grid filled with [길] is not allowed.)
      4) [물] grid and [흙] grid should not be adjacent to one another.
      5) [풀] should always be adjacent to [물] or [길].

Given the map, write a program that determines whether the map is "Good" or "Bad".

Input

First line of input has the number of types of tiles, N. (1 ≤ N ≤ 100,000). In the next N lines, each line has the information about the tiles. Tile information contains an integer K, which holds the specific ID of the tile, and information about the 3 x 3 grid. For convenience reasons, we will label G(풀), W(물), R(길), D(흙) and the grid information should be read from left to right, top to bottom. The only value that shouldn't be the same is K. The following image's ID is 283 and the grid information is shown. (1 ≤ K ≤ 100,000

283 G R D G R R W W W

After that, the two integers C, R which represent the size of the map are given. C is the number of tiles in one row, and R is the number of rows. (1 ≤ C, R ≤ 300). Following R lines represent the map with the ID numbers of each grid.

Output

Output the whether the given map is Good or Bad.

Input Example 1

       4
33 G G G R R G G R G
71 G R D G R R G R D
65 G R G W R R W W W
92 G R D R R R W W W
2 2
33 71
65 92

Output Example 1

        Bad
